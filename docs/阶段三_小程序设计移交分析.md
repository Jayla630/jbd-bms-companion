# 阶段三 · 小程序设计移交分析

来源：Claude Design 项目《BMS 小程序交互原型》（`BMS 小程序交互原型.dc.html`，可交互版）。
原型用 `data-page` / `data-component` / `data-field` 三层标注，本文档是它到原生小程序工程的映射契约。

## 一、页面（data-page → pages/）

| data-page | 页面 | 小程序路由 | 导航方式 |
|---|---|---|---|
| `overview` | 总览：SOC 大数字 + 电量条、三指标、单体电压、温度/MOS、保护概览入口；未连接时空态 | `pages/overview/overview` | tabBar ① |
| `control` | 控制：充电/放电 MOS、均衡三张开关卡 + 未连接/锁定提示 + 同口告警 | `pages/control/control` | tabBar ② |
| `protection` | 保护状态：锁定横幅、汇总卡、MOS 状态行、13 项 status-chip 网格、解锁弹层入口 | `pages/protection/protection` | tabBar ③ |
| `connect` | 设备连接：我的设备 + 附近设备扫描列表，带返回键、隐藏 tabBar | `pages/connect/connect` | `wx.navigateTo` 普通页 |

原型里的 `tab-bar` 组件（总览/控制/保护 + 保护红点）由 app.json 原生 `tabBar` 承接；
保护红点用 `wx.showTabBarRedDot({ index: 2 })` / `hideTabBarRedDot` 实现。

## 五、状态机（原型脚本已实现，移植语义）

- **link**：`disconnected → connecting → connected`，异常掉线 `dropped`（空态文案不同：「尚未连接」vs「连接已断开·可重连」）。
- **mode**：由 `current` 符号推导 —— 正=charge（绿）、负=discharge（蓝）、0=idle（灰）。
- **开关（回读落定）**：点击 → `pend[k]=target`（轨道浅蓝 + 转圈"等待回读确认"）→ ~1.3s 回读 → 落定并 toast；**无乐观更新**。锁定中（bit12=1）充/放开关跳保护页，均衡不受限。
- **unlock**：`idle → step1_send/ack → step2_send/ack → step3_send/ack → adjudicating → success | failed`。
  命令序列 0xE1：`0x0003 全关 → 0x0001 开充 → 0x0000 开放`；三步 ack 完成 ≠ 成功，**以回读 0x03 的 bit12 裁决**。失败可"从头重跑"（幂等可重入，与上位机切片 3 一致）。
- **压差**：`delta_mv > 40` 视为偏大（徽标转橙、最低柱转橙）。

## 六、落地约定

- 原生小程序（无 uniapp/taro），mock 数据先行：`services/mock-bms.js` 提供与未来
  BLE 层同形的接口（`scan/connect/disconnect/toggle/unlock`），页面只依赖
  `services/store.js` 的订阅，后续切换真实 BLE 不动页面层。
- 原型右列"演示状态切换器"与"移交说明"为设计辅助，不进小程序。
- tabBar 图标暂用纯文字（原生 tabBar iconPath 可选），后续补 81×81 PNG。
