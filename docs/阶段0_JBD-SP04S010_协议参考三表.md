## 〇、通用约定（写代码前先记住这几条）

| 项 | 约定 |
|---|---|
| 物理层 | RS485 / UART，波特率默认 **9600**（也可能是自定义率，真机核对） |
| 字节序 | 所有多字节整数一律 **大端**（高字节在前） |
| 起始 / 结束 | 每帧固定 `0xDD` 开头、`0xDD` … `0x77` 结尾 |
| 电压单位 | 总电压：**10 mV**（读数 ÷100 得伏特）；单体电压（0x04）：**1 mV**（读数 ÷1000 得伏特）—— 注意两者刻度不一样 |
| 电流单位 | **10 mA**，**有符号 s16**，正=充电、负=放电（JBD 约定，实测再确认符号方向） |
| 容量单位 | **10 mAh** |
| 温度 | u16，单位 **0.1 K**，摄氏度 = (原始值 − 2731) ÷ 10 |

### 校验算法（最容易对不上的地方，单独拎出来）

请求帧和响应帧的校验**覆盖范围不一样**，这是社区里最常见的踩坑点：

- **请求帧校验** = `(0x10000 − (寄存器字节 + 长度字节 + Σ数据字节)) & 0xFFFF`
- **响应帧校验** = `(0x10000 − (状态字节 + 长度字节 + Σ数据字节)) & 0xFFFF`
- 两边都**不把** `0xDD`、`0x77` 计入；
- 请求帧把"寄存器字节"算进去，响应帧把"状态字节"算进去（读成功时状态=0x00，所以响应数值上等于 长度+数据）；
- 别手滑把响应里的寄存器字节也加进去——那样就永远差一个寄存器值，怎么调都对不上。
- 结果高字节在前、低字节在后。

## 一、帧结构表

### 1.1 请求帧（上位机 → BMS）

| 字段 | 字节数 | 取值 | 说明 |
|---|---|---|---|
| 起始码 | 1 | `0xDD` | 固定 |
| 操作码 | 1 | `0xA5`=读 / `0x5A`=写 | |
| 寄存器 | 1 | 见寄存器表 | 要读/写的寄存器地址 |
| 长度 | 1 | 数据字节数 | 纯读命令为 `0x00` |
| 数据 | N | — | 读命令无数据；写命令放待写内容 |
| 校验 | 2 | 见上 | 高字节在前 |
| 结束码 | 1 | `0x77` | 固定 |

### 1.2 响应帧（BMS → 上位机）

| 字段 | 字节数 | 取值 | 说明 |
|---|---|---|---|
| 起始码 | 1 | `0xDD` | 固定 |
| 寄存器 | 1 | 回显请求的寄存器 | |
| 状态 | 1 | `0x00`=成功 / `0x80`=出错 | 未进配置模式就读配置寄存器会回 `0x80` |
| 长度 | 1 | 后续数据字节数 | |
| 数据 | N | — | 具体内容见寄存器表 |
| 校验 | 2 | 见上 | 高字节在前 |
| 结束码 | 1 | `0x77` | 固定 |

### 1.3 黄金测试向量（已逐字节验算，直接拿去当单元测试基准）

**读基础信息 · 请求：**
```
DD A5 03 00 FF FD 77
```
校验核对：0x10000 − (0x03 + 0x00) = 0xFFFD ✓

**读基础信息 · 响应（真实抓包，4 串 / 3 温感）：**
```
DD 03 00 1D 06 0B 00 00 01 ED 01 F4 00 00 2C 7C 00 00 00 00 10 00 80 63 02 04 03 0B A0 0B 9D 0B 98 FA 55 77
```
校验核对：长度 0x1D + Σ数据 = 1451 = 0x05AB，0x10000 − 0x05AB = 0xFA55 ✓

逐字段解读（写模拟器时这就是"正确答案"）：

| 字段 | 原始 | 解读 |
|---|---|---|
| 总电压 | `06 0B` | 15.47 V |
| 电流 | `00 00` | 0.00 A |
| 剩余容量 | `01 ED` | 4.93 Ah |
| 标称容量 | `01 F4` | 5.00 Ah |
| 循环次数 | `00 00` | 0 |
| 生产日期 | `2C 7C` | 2022-03-28 |
| 均衡状态(低/高) | `00 00 / 00 00` | 无均衡 |
| 保护状态 | `10 00` | bit12=1 → **MOS 软件锁定** |
| 软件版本 | `80` | 8.0 版 |
| 剩余电量(SOC) | `63` | 99 % |
| MOS 状态 | `02` | bit1=放电开、bit0=充电关 |
| 电池串数 | `04` | 4 串 |
| NTC 数量 | `03` | 3 个 |
| 温度1/2/3 | `0BA0 / 0B9D / 0B98` | 24.5 / 24.2 / 23.7 ℃ |

> 顺带一提：这帧里"保护状态 = 0x1000（bit12 MOS 软件锁定）"正好对应你摘要里那个已知坑——**"保护锁定解不开、充放开关切不动"**。真机遇到这个别慌，一般是切换顺序问题（先关充放两个，再按"充电、放电"顺序开），不是板子坏了。

## 二、寄存器表

### 2.1 读寄存器（操作码 `0xA5`）

| 寄存器 | 名称 | 数据内容 |
|---|---|---|
| `0x03` | 基础信息 | 见 2.4 逐字节展开 |
| `0x04` | 单体电压 | N × u16，大端，单位 **mV**（4 串就是 8 字节） |
| `0x05` | 设备名称 | ASCII 字符串（如 `LH-SP04S…`） |

### 2.2 写 / 控制寄存器（操作码 `0x5A`）

| 寄存器 | 名称 | 报文 / 说明 |
|---|---|---|
| `0x00` | 进入工厂(配置)模式 | `DD 5A 00 02 56 78 FF 30 77`（向 0x00 写入魔数 0x5678）。**不先进这个模式，读配置寄存器会回 0x80** |
| `0x01` | 退出并保存 | 向 0x01 写入后配置固化；也用于清错误记录 |
| `0xAA` | 错误(保护触发)计数 | 各类保护累计触发次数，开机读一次可复盘历史 |
| `0xE1` | MOSFET 控制 | 写 u16：**bit0=1 关放电，bit1=1 关充电**；`0x0000`=全开、`0x0003`=全关。例：`DD 5A E1 02 00 03 FF 1A 77` 全关。⚠️ 语义与"MOS 状态位"相反（状态位是 1=开），别搞混 |
| `0xE2` | 均衡开关 | 使能/关闭主动均衡 |

> **配置模式读法**：进模式后用 `0xA5` 依次读 `0x10`~`0xA2` 及 `0xAA`，即可把 JBDTOOLS 里那一整屏参数抠出来。真机到手用这个办法把"保护参数表"的真实地址与默认值一次性抓全。

### 2.3 保护状态位（0x03 里的"保护状态" u16，bit=1 表示该保护已触发）

| bit | 含义 | bit | 含义 |
|---|---|---|---|
| 0 | 单体过压 | 7 | 放电低温 |
| 1 | 单体欠压 | 8 | 充电过流 |
| 2 | 整组过压 | 9 | 放电过流 |
| 3 | 整组欠压 | 10 | 短路 |
| 4 | 充电过温 | 11 | 前端采集 IC 错误 |
| 5 | 充电低温 | 12 | **MOS 软件锁定** |
| 6 | 放电过温 | 13~15 | 保留 |

| 偏移 | 字节 | 字段 | 换算 |
|---|---|---|---|
| 0 | 2 | 总电压 | ×10 mV |

| 14 | 2 | 均衡状态(17~32 串) | 位图 |

## 四、真机到手后的校准清单（阶段〇→阶段一之间做）

1. 双链路验货：小象 App（蓝牙）能连、JBDTOOLS（485）能读，两边数据对得上。
2. 用 JBDTOOLS 读一遍配置，把上表"查真机"那些寄存器的**真实地址、默认值、单位刻度**全部填实。
3. 抓几帧真实 0x03/0x04/0x05 报文存档，补进"黄金测试向量"，作为上位机与模拟器的共同基准。
4. 确认波特率、串数（应为 4）、NTC 路数、电流符号方向（充正放负还是反过来）。
5. 复现一次"MOS 软件锁定"的解锁流程，记录正确的开关切换顺序，写进上位机的操作提示。
